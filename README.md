# KKX — 外贸获客智能体

> 为 SWSAGE/WELP 打造的 AI 驱动 B2B 海外客户开发系统

---

## 一、这是什么？

KKX 是一套**自动化海外客户开发工具**，解决外贸业务团队在获客环节的三大痛点：

1. **找客户难** — 手动搜索效率低、覆盖窄、质量不稳定
2. **判断慢** — 面对大量线索难以快速判断哪些值得跟进
3. **跟进慢** — 从发现客户到发出第一封开发信周期太长

系统能在 **3-5 分钟内**自动完成：搜索目标市场真实客户 → 评分分级 → 补充联系方式 → 生成英文开发信草稿 → 同步到飞书表格，直接交给业务员跟进。

---

## 二、能做什么？

### 自动发现目标客户

系统按照预设的**产品关键词**和**目标市场**，在 Google 上自动搜索真实客户公司：

- **产品范围**：陶瓷杯、不锈钢杯、软木餐垫、美耐皿托盘等礼品/家居类产品
- **目标市场**：英国（首选）→ 澳洲（重点）→ 美国（培育）
- **客户类型**：礼品零售商、百货买手、家居连锁、家居批发商等

### 智能筛选与分级

系统不是简单地把搜索结果堆给你，而是通过两层评分自动分级：

| 等级 | 含义 | 业务员动作 |
|------|------|-----------|
| **A** | 高度匹配，有联系方式 | 立即跟进 |
| **B** | 比较匹配，联系方式部分完整 | 发送开发信 |
| **C** | 边缘匹配，信息待补充 | 继续背调 |
| **D** | 匹配度低 | 暂不优先 |

> **关键设计**：A/B/C/D 等级只由"业务匹配度"决定，与联系方式是否完整无关。联系方式不全的高价值客户不会被误判为 D 级，而是进入"继续背调"流程。

### 自动补充联系方式

对 A/B 级和高潜 C 级客户，系统自动进行**二次搜索**，补充：

- 官网 contact/about 页面
- 真实邮箱（非伪造）
- 决策人姓名和职位（如 Category Buyer、Purchasing Manager）
- LinkedIn 个人页

### 自动生成英文开发信

对符合条件的客户（A/B 级、有真实联系方式），系统调用大语言模型自动生成**个性化英文开发信**，而非千篇一律的模板邮件。

### 一键同步到飞书

每次运行结果自动写入飞书多维表格，包含 15 个业务字段：

| 字段 | 说明 |
|------|------|
| 公司名称 | 真实客户公司名 |
| 官网 | 公司官方网站 |
| 国家 | 从页面内容提取（英国/澳洲/美国） |
| 客户类型 | 零售/批发/连锁等 |
| 客户等级 | A / B / C / D |
| 客户符合度分 | 0-100 分 |
| 分级原因 | 为什么这个等级 |
| 关键判断信号 | 市场/类型/产品/风险信号 |
| 推荐联系人 | 决策人姓名 + 职位 |
| 联系方式线索 | 邮箱/电话/LinkedIn |
| 邮件草稿 | 自动生成的英文开发信 |
| 搜索处理状态 | 已补全可跟进 / 需第三次搜索 / 继续背调 / 暂不优先 |
| 下一步动作 | 系统建议的下一步操作 |
| 备注 | 补充说明 |
| 创建时间 | 数据生成时间 |

## 三、工作流程

```
业务员点击"开始运行"
        │
        ▼
   ┌─────────────┐
   │  1. 加载配置  │  公司画像 + 评分规则（业务员可调整）
   └──────┬──────┘
          ▼
   ┌─────────────┐
   │  2. 搜索客户  │  按 UK/AU/US 市场分别搜索，纯英文查询
   └──────┬──────┘
          ▼
   ┌─────────────┐
   │  3. 智能过滤  │  自动排除目录页/资源页/文章页/电商平台
   └──────┬──────┘
          ▼
   ┌─────────────┐
   │  4. 匹配评分  │  市场匹配 + 客户类型 + 产品场景 + 风险评估
   └──────┬──────┘
          ▼
   ┌─────────────┐
   │  5. 联系人补全 │  对高优客户二次搜索：邮箱/电话/LinkedIn
   └──────┬──────┘
          ▼
   ┌─────────────┐
   │  6. 邮件草稿  │  LLM 生成个性化英文开发信
   └──────┬──────┘
          ▼
   ┌─────────────┐
   │  7. 同步飞书  │  自动建表，15 个业务字段，直接可用
   └─────────────┘
```

全程约 **3-5 分钟**，产出 10-20 条经过筛选和评分的真实客户线索。

## 四、核心业务优势

### 与传统手动搜索对比

| 对比项 | 手动搜索 | KKX 智能体 |
|--------|---------|-----------|
| 覆盖范围 | 业务员已知渠道 | Google 全量搜索 |
| 筛选质量 | 依赖个人经验 | 系统规则 + 自动过滤 |
| 分级效率 | 逐条判断 | 自动评分分级 |
| 联系人补充 | 逐家查找 | 自动二次搜索 |
| 开发信 | 手动写或套模板 | AI 个性化生成 |
| 单次耗时 | 2-3 小时 | 3-5 分钟 |
| 产出质量 | 不稳定 | 标准化输出 |

### 质量保障

系统内置了多重质量保障机制：

1. **只找真实公司页** — 自动排除展会目录页、主办方资源页、文章博客页、电商平台商品页
2. **国家从内容提取** — 不从域名后缀硬猜，从页面正文中的地址/公司简介提取
3. **不伪造联系方式** — 所有邮箱/电话/LinkedIn 均为真实搜索结果，不拼接假链接
4. **高等级不降档** — 客户匹配等级与联系方式完整度无关，缺联系人的高价值客户不会被误判为 D 级

## 五、快速开始

### 前置条件

- Serper API Key（Google 搜索 API，[serper.dev](https://serper.dev)，每月 2500 次免费）
- 飞书开放平台应用凭证
- 大语言模型 API Key（DashScope / OpenAI 兼容接口）

### 配置环境变量

编辑 `simple_backend_project_root/.env`：

```bash
# 搜索配置
SEARCH_PROVIDER=serper
SERPER_API_KEY=你的serper密钥

# 飞书配置
FEISHU_APP_ID=你的飞书应用ID
FEISHU_APP_SECRET=你的飞书应用密钥
FEISHU_BITABLE_APP_TOKEN=你的飞书多维表格AppToken

# LLM 配置
DASHSCOPE_API_KEY=你的dashscope密钥
OPENAI_MODEL=qwen3.5-plus
```

### 启动服务

```bash
# 后端（API 服务器）
cd simple_backend_project_root
python -m uvicorn api_server:app --host 0.0.0.0 --port 8001

# 前端（工作台界面）
cd web
npm install && npm run dev
```

访问 http://localhost:3000 打开工作台，点击"开始运行"即可。

## 六、飞书表格说明

每次运行会在飞书 Base App 下**自动创建一张新表**：

- **表名**：`客户表_MM-DD_HH-mm-ss`（如 `客户表_04-13_10-28-18`）
- **防冲突**：如果重名自动追加 `_2` / `_3`，不会因重名中断
- **字段**：15 个业务字段自动创建，开箱即用
- **数据**：每条 lead 包含完整的评分、分级、联系方式和邮件草稿

### 飞书表格使用建议

1. **每次运行独立一张表** — 方便对比不同批次的结果
2. **A/B 级优先跟进** — 按"客户等级"排序，优先处理高分客户
3. **C 级持续培育** — 标记为"继续背调"，后续运行中可能升级
4. **利用"下一步动作"字段** — 系统已为每条线索标注建议动作

## 七、如何调整业务策略

系统的核心业务逻辑由以下两个 JSON 文件控制，业务员可自行调整：

| 文件 | 用途 | 可调参数 |
|------|------|---------|
| `swsage_company_profile_v3.json` | 公司画像 | 产品关键词、目标市场、目标客户类型、竞争优势 |
| `swsage_grading_rules_v3.json` | 评分规则 | 各维度权重、分级阈值、下一步动作映射 |

### 常见调整场景

**切换目标市场** — 修改 `swsage_company_profile_v3.json` 中的 `target_markets`

**增加新产品线** — 在产品关键词池中添加新词

**调整分级严格度** — 修改 `swsage_grading_rules_v3.json` 中的阈值

                                    ┌─────────────┼─────────────┐
